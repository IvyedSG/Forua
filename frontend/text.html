Is it possible to move from (a,b) to (c,d)

Asked 4 years, 11 months ago

Modified today

Viewed 118 times

The problem was to output whether it is possible to move from a given point (a,b) to target (c,d).

(a,b) -> (a+b,b) 
  (a,b) -> (a,a+b)
  
  For instance, (1,3) to (4,10) is True. You can do the following: Using 2nd move 3 times, (1,3) -> (4,3) -> (4,7) -> (4,10). Using 1st move 1 time, (1,3) -> (4,3).
  
  I came up with the following recursive method:
  
  
  def move(a,b,c,d):
  if (a,b) == (c,d):
      return True
  elif a > c or b > d:
      return False
  else:
      if move(a+b,b,c,d):
          return True
      if move(a,a+b,c,d):
          return True
  return False

If all the numbers have to be positive, then I believe there is a much quicker solution.

Trying to find if we can get from (a,b) to say (34,12), then we can do the only way with positive numbers to reach (34, 12) is to apply the second rule to (34, 12) (34, 12). The only way to get (34, 12) is to apply the second rule to (34, 12). The only way to get (34, 12) is to apply the first rule to (34, 12). So an algorithm is pretty simple: Loop on (c,d), replacing it with (c-d, d) or (c, d-c) and stopping when you hit a match, when one of c or d is 0.
